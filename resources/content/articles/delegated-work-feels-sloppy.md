# 任せた相手の雑さが引っかかって、結局自分で回収したくなる時のレシピ

::me
人に任せればいいのに、
任せたあとずっと気になってしまいます。

途中で口を出したくなるし、
ちゃんとされなかった時のことを思うと、
最初から自分でやったほうが早い気がする。

人が嫌いなわけじゃない。
むしろ好きだし、
できれば気持ちよく頼りたい。

でも、
任せると落ち着かないんです。
::

それは、
人を信頼できないというより、
**任せたあとに、自分の中が乱れるのがつらい**
状態かもしれません。

待っている間に頭を持っていかれる。
途中で口を出したくなる。
ちゃんとされなかった時の回収コストまで、
先に自分が背負ってしまう。

だから、
任せないほうが静かでいられるように見えるんですね。

::me
そうそう。

しかも、
相手を責めたいわけじゃないのよ。
責めたくないから、
静かに自分で回収してしまう感じ。

でもそれを続けてると、
結局ずっと私が持つことになる。
::

そうですよね。

ここで起きているのは、
ただの段取りの問題ではありません。

本当に痛いのは、
相手が雑だった時に
**私は大事にされてないのかも**
が立ち上がることではないですか。

::me
ああ、それだ。

こちらは誠心誠意でいるのに、
雑に返されると、
なめられてるのか？みたいな感じが出る。

そこで私の中の[ざっちー](/articles/what-is-zatchy)がすごく暴れる。
だから最初から心を閉じてるのかもしれない。
::

そこが大きなポイントです。

あなたは最初から冷たいわけでも、
人を見下しているわけでもありません。

むしろ逆で、
**誠心誠意でいたい人**
なんです。

だからこそ、
雑に返された時に傷が深い。

その痛みを何度も味わいたくないから、
期待を下げる。
任せない。
最初から諦めておく。

そうすると一見平和ですが、
本当はずっと、
自分のほうが先に心を閉じる運用になってしまいます。

::me
じゃあ、どうしたらいいの。

もっと人を信じればいいって話ではないよね。
::

はい。
無理に信じる話ではありません。

先にやることは、
雑さを受けた瞬間に
**雑だった**
ことと
**私は大事にされていない**
ことを、
すぐ一つにしないことです。

この二つがくっつくと、
ざっちーが一気に暴れて、
相手の雑さ以上の傷になります。

だからまずは、

「いま私は、相手の雑さそのものに傷ついているのか」
「それとも、大事にされていないところまで一気に飛んでいるのか」

を分けて見てください。

::me
たしかに、
そこが一緒になると一気に苦しくなる。
::

そうなんです。

そして、
すぐ回収役に戻らないことも大事です。

回収したくなるのは、
整えたいからです。
これ以上ざわつきたくないからです。

でも、
毎回すぐ自分で回収すると、
「やっぱり任せられない」が強化されていきます。

だから、
すぐ全部を背負い直さなくていい。

先にやるのは、
回収ではなく、
**自分のざっちーが暴れていると気づくこと**
です。

そのうえで、
相手との関係を壊さずに、
自分も壊さない任せ方を作っていく。

丸投げするか、
全部自分で持つかの二択ではありません。

頼む範囲を小さくする。
途中確認を前提にする。
最初から期待値を下げるのではなく、
**任せ方を調整する**んです。

::point
## 任せたあと、雑さで心を閉じたくなる時のレシピ
* 任せられないのではなく、任せたあとの自分の乱れがつらいことがある
* 相手の雑さと「私は大事にされてない」をすぐ一つにしない
* [ざっちー](/articles/what-is-zatchy)が暴れたら、まず回収より先に「反応が起きている」と気づく
* 関係を壊さないために全部抱えるのではなく、自分が壊れない任せ方に切り直す
* 誠心誠意の自分と尊厳を守りながら、人に任せる道は作れる
::
